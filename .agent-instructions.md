## TOKEN OPTIMIZATION PROFILE
Source: .agent-context/state/token-optimization.json

Token optimization mode is enabled for agent: copilot.
Preferred shell proxy: native-fallback.

Apply command rewrites before running verbose shell commands:
- ls -la => rtk ls . (Directory listings are grouped and compacted before entering the model context.)
- tree => rtk ls . (Large trees are summarized without losing high-signal structure.)
- cat file.ext => rtk read file.ext (File reads keep signatures and relevant lines while trimming boilerplate.)
- rg pattern . => rtk grep "pattern" . (Search hits are grouped and deduplicated for faster reasoning.)
- git status => rtk git status (Status output is condensed to actionable file changes.)
- git diff => rtk git diff (Diff noise is trimmed while preserving code-review context.)
- git log -n 10 => rtk git log -n 10 (History is converted to compact one-line summaries.)
- npm test => rtk test npm test (Test results emphasize failures and suppress repetitive pass logs.)
- npm run build => rtk err npm run build (Build runs return errors and warnings only for quicker remediation.)
- tsc --noEmit => rtk tsc (TypeScript diagnostics are grouped by file with less repetition.)
- eslint . => rtk lint (Lint output is grouped by rule and location to reduce token churn.)
- docker ps => rtk docker ps (Container state is summarized into compact rows.)
- kubectl get pods => rtk kubectl pods (Kubernetes output is condensed and stable for agent consumption.)

Important scope note:
- Shell rewrite hooks affect shell tool calls only.
- Built-in read/grep/glob style tools may bypass shell rewrites, so explicit compact shell commands should be preferred in high-volume sessions.

Output folding policy:
- Mode: compact-high-signal-output.
- Preserve always: exit-code, failing-command, file-and-line, error-message, actionable-warning.
- Fold by default: repeated-success-lines, large-unchanged-diff-context, verbose-install-progress, duplicated-stack-frames, low-signal-directory-noise.
- Safety boundary: never hide failing checks, never drop the command that produced output, never persist secrets into summaries, hydrate full output only when the compact summary is insufficient.

Fallback behavior when external proxy is unavailable:
- Use command variants with bounded output such as git diff --stat, rg --max-count, and npm test -- --reporter=dot.
- Request only the lines or sections required for the current decision.
- If shell output is still large, summarize and continue iteratively instead of dumping full logs.
